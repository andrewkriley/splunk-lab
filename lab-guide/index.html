Step 1

Set Up the Lab

Get Splunk Enterprise running locally in under five minutes using Docker Compose. No Splunk licence is required — the free trial tier covers everything in this lab.

Prerequisites

🐳
Docker Desktop
Install and make sure it's running before you start.
📦
Node.js
Required on the host for npx when connecting Claude to MCP (Step 3).
🤖
Claude Code or Claude Desktop
For AI-driven MCP from your editor or desktop app. Optional if you only use Ask Splunk in the browser.

Installation

  1. Clone the repository
    bash
    git clone https://github.com/andrewkriley/splunk-lab.git cd splunk-lab
  2. Configure your password

    Copy the example environment file and set a password for the Splunk admin account.

    bash
    cp .env.example .env

    Open .env and set your password:

    .env
    SPLUNK_PASSWORD=YourPassword123
    ℹ️ Password must be at least 8 characters and contain both letters and numbers.
  3. Start the stack
    bash
    docker compose up -d

    Splunk takes 60–90 seconds to initialise on first boot. Watch the logs:

    bash
    docker compose logs -f splunk

    When you see Ansible playbook complete, Splunk is ready.

  4. Open Splunk

    Navigate to http://localhost:8000 and log in with:

    Usernameadmin Passwordthe value you set in .env
    ℹ️ Before you run Buttercup searches in Search & Reporting, set the time range to All time (sample data is dated January 2025; Last 24 hours usually shows no events). Step 2 copies SPL that includes earliest=0 latest=now so results work regardless of the picker.
Step 2

Explore Buttercup Data

The lab ships with Buttercup Games sample data — a fictional online game store. Three datasets are automatically indexed at startup with no manual import needed.

What's in the data

Sourcetype Description Index
buttercup_web Web storefront access logs (Apache Combined format) buttercup
buttercup_sales Vendor sales transactions — units, revenue, product buttercup
buttercup_products Product catalogue with prices and categories buttercup

Try these searches

Open Search & Reporting in Splunk and run the queries below. The sample data is from January 2025; Splunk’s default time range is often “last 24 hours,” which would show no events. Each query includes earliest=0 latest=now so it matches all indexed data — or set the picker to All time and you can omit those terms.

Traffic by HTTP status code
index=buttercup earliest=0 latest=now sourcetype=buttercup_web | stats count by status
Sales revenue by vendor
index=buttercup earliest=0 latest=now sourcetype=buttercup_sales | stats sum(revenue) as total_revenue by vendor | sort -total_revenue
Top products by units sold
index=buttercup earliest=0 latest=now sourcetype=buttercup_sales | stats sum(units_sold) as total_units by product | sort -total_units
Revenue trend over time
index=buttercup earliest=0 latest=now sourcetype=buttercup_sales | timechart span=1d sum(revenue) by vendor
Step 3

Connect to MCP

🔒 The MCP server has no authentication and listens on 127.0.0.1:8050 only — fine for this local lab, but do not expose that port to untrusted networks.

The Splunk MCP server runs alongside Splunk and exposes tools for search, dashboards, and alerts over the Model Context Protocol. Use Ask Splunk in the sidebar for a browser UI, or connect Claude Code / Claude Desktop using the tabs below.

Claude Code
Claude Desktop

No setup required

The project ships with .claude/settings.json pre-configured. Open Claude Code in the splunk-lab directory with the stack running — the Splunk MCP tools are automatically available.

json
{ "mcpServers": { "splunk-lab-guide": { "command": "npx", "args": ["-y", "mcp-remote@0.1.38", "http://localhost:8050/sse"] } } }

Requires Node.js on the host for npx. The MCP server uses mcp-remote as a local proxy — Claude Code communicates over stdio, avoiding the HTTPS requirement for direct SSE connections.

Try these prompts

  • Search Splunk for the top 5 vendors by revenue
  • Show me HTTP error rates from the Buttercup web logs
  • Create a dashboard showing web traffic by status code
  • What products have the highest units sold this month?

Add the MCP server

Open your Claude Desktop config file and add the mcpServers block:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json
json
{ "mcpServers": { "splunk-lab-guide": { "command": "npx", "args": ["-y", "mcp-remote@0.1.38", "http://localhost:8050/sse"] } } }

Restart Claude Desktop — the Splunk tools will appear in the tool list whenever the lab stack is running.

Requires Node.js on the host for npx. Claude Desktop expects HTTPS for direct SSE to remote URLs; this lab uses plain HTTP on localhost, so mcp-remote proxies over stdio the same way as in Claude Code.

Try these prompts

  • Search Splunk for the top 5 vendors by revenue
  • Show me HTTP error rates from the Buttercup web logs
  • Create a dashboard showing web traffic by status code
  • What products have the highest units sold this month?

Documentation

Architecture, configuration reference, and guides for the Splunk Lab.

Architecture

Four public-facing Docker Compose services on the host (Splunk, MCP, lab guide, status sidecar). The Ask Splunk backend runs internally and is reached through the lab guide. All host ports are bound to 127.0.0.1 — accessible from your machine only.

ServiceImage / BuildPortRole
splunk splunk/splunk:10.2.1 8000 8088 8089 Splunk Enterprise — Web UI, HEC, REST API. Buttercup sample data auto-indexed on boot.
splunk-mcp mcp/Dockerfile 8050 MCP server (SSE). Bridges Claude / Ask Splunk to Splunk search, indexes, and saved searches.
lab-guide nginx:alpine ${LAB_GUIDE_PORT} This guide — static HTML plus nginx proxies: /api/status → status-api, /ask/api/* → Ask Splunk backend (FastAPI on chat:3000, internal only).
chat chat/Dockerfile (internal) Ask Splunk FastAPI — MCP Tool Explorer + optional Chat. Not exposed on the host; use Lab Tools → Ask Splunk here or open /ask/ under the lab guide URL.
status-api status-api/Dockerfile 8081 (internal) Health-check sidecar. Probes container states and service endpoints via Docker API.

Data flow

Browser ──▶ Lab guide (/ask/) ──▶ Ask Splunk API (chat) ──▶ MCP (:8050) ──▶ Splunk (:8089) │ │ │ └──▶ Anthropic API (Chat tab only, when API key set) │ ┌────────────────────┼────────────────────┐ ▼ ▼ ▼ buttercup_web buttercup_sales buttercup_products

Networking

Containers communicate on the splunk-lab_default bridge network using service names as hostnames (e.g. splunk-mcp connects to splunk:8089). Host port bindings are 127.0.0.1-only — not accessible from other machines. No authentication on the MCP endpoint by design.

Volumes

Volume / MountPurpose
splunk-varNamed volume — persists Splunk indexes and data across restarts. Deleted on docker compose down -v.
./buttercup_appBind mount — Buttercup sample data app auto-loaded by Splunk at boot.
./lab-guideBind mount (read-only) — this lab guide, served by nginx.
/var/run/docker.sockBind mount (read-only) — gives status-api read access to container states.

Configuration Reference

All configuration lives in .env (gitignored). Copy .env.example to .env before starting. Variables are read by Docker Compose at startup.

Required

VariableDefaultDescription
SPLUNK_PASSWORD Chang3d! Splunk admin password. Must be ≥8 chars with letters + numbers.

Optional

VariableDefaultDescription
SPLUNK_HEC_TOKEN a8b4c2d6-… Pre-set HEC token. Any UUID works — set before boot so you can send events immediately.
LAB_GUIDE_PORT 3131 Host port for this lab guide (includes Ask Splunk at /ask/).
ANTHROPIC_API_KEY unset Enables the Chat tab in Ask Splunk. The Explore Tools tab works without it.

MCP server environment

Set automatically by Docker Compose. Only change if you know what you're doing.

VariableValueNotes
SPLUNK_HOSTsplunkHostname only — the MCP client prepends https://.
SPLUNK_PORT8089Splunk REST API port (internal).
VERIFY_SSLfalseSplunk uses a self-signed cert locally.
TRANSPORTsseMCP transport mode. SSE is required for web clients.

Common commands

Start the lab
docker compose up -d
Watch Splunk startup (ready when "Ansible playbook complete" appears)
docker compose logs -f splunk
Stop containers (data preserved)
docker compose down
Full reset — wipe all indexed data
docker compose down -v
Run integration tests
pip install -r requirements-test.txt && pytest tests/ -v
